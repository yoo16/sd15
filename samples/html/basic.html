POST/GET の違い

GET

URLにキーや値を記述して送信

Formを実行するとパラメータを追加した URI で送信


        http://xxxx.com?item_id=xxxx&user_id=xxxx
  • 複数の場合は「&」で繋ぐ
  • key=value の形式でパラメータを設定
  • 複数パラメータの場合は「&」で繋ぐ

特徴

  • パラメータを指定してページやデータを取得する場合に向いている
  • パーマネントリンク（永久的なリンク）に向いている
  • Webシステムにおいて作成や削除の処理する場合、直接URLを記述して実行されるため危険
  • パスワードもそのままURLに記述される

GETサンプル

POST

URLにデータ記述せず、データを隠蔽して送信


        http://xxxx.com

特徴

  • データを内部的に隠蔽して送信できる
  • パラメータをURIに記述しないため、常に同じURL
  • 文字制限に事実上制限がない
  • URL配布に向かない

REST(REpresentational State Transfer)

Webサービスにおいて、URIからHTTPメソッドでアクセスすることでデータの送受信する

RESTfulなサービス

  • アドレス指定可能なURIで公開されていること
  • インターフェース(HTTPメソッド)の統一がされていること
  • ステートレスであること
  • 処理結果がHTTPステータスコードで通知されること

URI と URL の違い

URI

Uniform Resource Identifier の略で、名前または場所を識別するルールの総称

URL

URIにおいて、場所(Location)を表すもの
